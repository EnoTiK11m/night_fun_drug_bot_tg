# Руководство по эксплуатации

[![ENG](https://img.shields.io/badge/lang-ENG-0078D4?logo=googletranslate&logoColor=white)](PRODUCTION.en.md)

## Рекомендуемый способ запуска

Для развёртывания на одном сервере используйте Docker Compose:

```bash
docker compose up -d --build
docker compose logs -f bot
```

Согласно конфигурации Compose, база данных SQLite хранится в каталоге
`./data`, а журналы работы — в `./logs`. Оба каталога необходимо включить
в резервное копирование и не следует добавлять в Git.

## Обязательные переменные окружения

Создайте файл `.env` на основе `.env.example` и задайте:

- `BOT_TOKEN`;
- `API_USER_ID`;
- `API_KEY`;
- `ADMIN_USER_IDS`.

Переменная `DB_PATH` задаётся в `docker-compose.yml` и указывает на
`/app/data/bot_data.db`.

Дополнительные настройки доступа:

- `ALLOWED_USER_IDS` — Telegram ID пользователей через запятую. Пустое
  значение разрешает пользоваться ботом всем пользователям в личных чатах;
- `ALLOWED_CHAT_IDS` — ID чатов через запятую, в которых разрешена работа бота;
- `ALLOW_GROUP_CHATS` — установите `true`, только если намеренно разрешаете
  использование бота в группах.

Администраторам из `ADMIN_USER_IDS` доступ разрешён всегда.

## Резервное копирование

Для создания резервной копии работающей базы рекомендуется использовать
вспомогательный скрипт:

```bash
python scripts/backup_sqlite.py --db data/bot_data.db --output-dir backups
```

Если файлы копируются вручную, сначала остановите контейнер. Как минимум
сохраните:

- `data/bot_data.db`;
- `data/bot_data.db-wal`;
- `data/bot_data.db-shm`.

## Локальный запуск в Windows

Для локального запуска в Windows можно использовать `rule34.bat`. Скрипт
завершает работу при штатной остановке, немедленно запускает бота заново после
команды `/restart` и перезапускает его через 10 секунд после неожиданного
завершения с ненулевым кодом.

## Рекомендации по эксплуатации

- Запускайте только один процесс бота для одной базы данных SQLite.
- Настройте ротацию и срок хранения каталога `logs/` согласно правилам сервера.
- Храните `.env` и резервные копии базы в закрытом доступе.
- При публичном использовании явно укажите возрастное ограничение для контента
  и разрешайте доступ только нужным пользователям и чатам.
- Следите за периодическими heartbeat-записями в `logs/info.log`. Их отсутствие
  обычно означает, что процесс бота остановлен или завис до запуска цикла
  обработки событий.
- Для перевода тегов требуется исходящий HTTPS-доступ к Google Translate.
  Перевод можно отключить через `TAG_TRANSLATION_ENABLED=false`; сохранённые
  английские теги, blacklist, поиск и подписки продолжат работать.
