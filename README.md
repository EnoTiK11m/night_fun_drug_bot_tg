# Rule34 Telegram Bot

Telegram-бот для поиска постов по тегам на Rule34, управления blacklist, подписками, историей запросов, избранным и настройками подписей.

## Версия 1.1

Релиз `1.1` посвящен стабильности бота и более понятному поведению для пользователя:

- добавлена пагинация избранного по 10 записей на страницу;
- исправлены сбои открытия избранного при большом количестве сохраненных постов;
- добавлены повторные попытки отправки медиа;
- если Telegram не принимает файл, пользователь получает понятное уведомление и ссылку на файл;
- поиск обходит страницы Rule34 API до конца выдачи;
- исправлены повторы постов при разных типах `post_id`;
- добавлен умный режим ожидания для исчерпанных подписок;
- подписки без новых постов проверяются реже и автоматически возвращаются к обычному интервалу после появления нового поста;
- добавлены SQLite-индексы для частых запросов.

## Возможности

- Поиск изображений, GIF и видео по тегам.
- Кнопка "Еще" для продолжения текущего поиска без повторов.
- Пользовательский черный список тегов.
- История последних поисков.
- Избранное с постраничным просмотром.
- Автоматические подписки на поисковые запросы.
- Умное увеличение интервала проверок для подписок, где все посты уже были отправлены.
- Настройки подписи под медиа.
- Автодополнение тегов через `/tags`.
- Поиск конкретного поста через `/id`.

## Установка

Требуется Python 3.11+.

```bash
pip install -r requirements.txt
```

Создайте `.env` по примеру:

```env
BOT_TOKEN=telegram_bot_token
API_USER_ID=rule34_user_id
API_KEY=rule34_api_key
SEARCH_COOLDOWN_SECONDS=3
```

Запуск:

```bash
python bot.py
```

На Windows также можно использовать:

```bat
rule34.bat
```

## Команды

- `/start` - открыть главное меню.
- `/search <теги>` - выполнить поиск.
- `/blacklist` - управление blacklist.
- `/subscriptions` - управление подписками.
- `/history` - история запросов.
- `/favorites` - избранное.
- `/settings` - настройки подписи.
- `/tags <запрос>` - подсказки тегов.
- `/id <post_id>` - открыть пост по ID.

## Поиск

Примеры запросов:

```text
genshin_impact
blue_eyes blonde_hair
1girl solo
```

Для тегов из нескольких слов используйте подчеркивания:

```text
large_breasts
short_hair
looking_at_viewer
```

Бот сохраняет отправленные `post_id` в SQLite и исключает их из следующих результатов для пользователя.

## Подписки

Подписка отправляет новые посты по сохраненному поисковому запросу с выбранным интервалом от 1 до 120 минут.

Если по тегу все доступные посты уже отправлены и новых постов на сайте нет, подписка не удаляется. Она переходит в режим ожидания:

- первая пустая проверка: следующая проверка примерно через 60 минут;
- вторая: через 120 минут;
- третья: через 240 минут;
- четвертая: через 480 минут;
- дальше: максимум через 720 минут.

Когда новый пост появляется, счетчик пустых проверок сбрасывается, и подписка возвращается к обычному интервалу.

## Избранное

Избранное открывается постранично по 10 постов. Это защищает меню от переполнения Telegram-сообщения и встроенной клавиатуры.

Доступны:

- просмотр списка;
- открытие отдельного поста;
- просмотр всех сохраненных постов галереей;
- удаление поста из избранного.

## Отправка медиа

Бот поддерживает:

- изображения через `reply_photo`;
- видео `.mp4` и `.webm` через `reply_video`;
- GIF через `reply_animation`.

Если Telegram не принимает файл из-за размера, формата, таймаута, сети или ошибки источника, бот:

1. повторяет отправку;
2. логирует ошибку;
3. отправляет пользователю уведомление и ссылку на файл.

## База данных

Используется SQLite-файл `bot_data.db`.

Основные таблицы:

- `users`;
- `blacklist`;
- `subscriptions`;
- `search_history`;
- `favorites`;
- `sent_posts`;
- `subscription_posts`;
- `user_settings`.
