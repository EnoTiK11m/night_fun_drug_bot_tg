# Rule34 Telegram Bot

Telegram-бот для поиска постов по тегам на Rule34, управления blacklist,
подписками, историей запросов, избранным и настройками подписей.

## Что умеет бот

### Поиск картинок и медиа

Бот ищет посты по тегам, которые вводит пользователь. Теги можно отправить:

- обычным сообщением;
- через кнопку `Поиск`;
- через команду `/search <теги>`.

Примеры запросов:

```text
genshin_impact
blue_eyes blonde_hair
1girl solo
```

Если тег состоит из нескольких слов, нужно использовать подчёркивание:

```text
large_breasts
short_hair
looking_at_viewer
```

Бот умеет отправлять разные типы файлов:

- изображения через `reply_photo`;
- видео `.mp4` и `.webm` через `reply_video`;
- GIF через `reply_animation`.

После отправки поста бот добавляет кнопки:

- `Подписаться` - создать подписку на текущий запрос;
- `Ещё` - найти следующий пост по этому же запросу;
- `Новый поиск` - начать новый поиск;
- `В избранное` - сохранить пост;
- `Открыть на сайте` - открыть пост на Rule34.

### Поиск без повторов

Бот запоминает ID постов, которые уже отправлялись конкретному пользователю.
При следующих поисках эти ID исключаются из результатов.

Это работает и для обычного поиска, и для кнопки `Ещё`, и для автоматических
подписок. Данные хранятся в таблице `sent_posts` в SQLite-базе.

Поиск больше не ограничен случайными страницами `0-10`. Бот начинает с первой
страницы результатов и идёт дальше, пока API не вернёт пустую страницу.

### Blacklist

У каждого пользователя есть свой blacklist тегов.

Через меню `Blacklist` можно:

- добавить один или несколько тегов;
- удалить один или несколько тегов;
- посмотреть текущий список.

Теги из blacklist автоматически добавляются к поисковому запросу как исключения.
Например, если пользователь ищет:

```text
genshin_impact
```

а в blacklist есть:

```text
male
furry
```

то итоговый запрос к API будет примерно таким:

```text
genshin_impact -male -furry
```

Также есть общий `DEFAULT_BLACKLIST` в `config.py`.

### Подписки

Бот умеет автоматически отправлять новые посты по расписанию.

Подписку можно создать:

- через меню `Подписки`;
- через кнопку `Подписаться` под найденным постом;
- на текущий поисковый запрос;
- на новый запрос.

Для подписок доступно управление:

- посмотреть список подписок;
- удалить подписку;
- поставить подписку на паузу;
- возобновить подписку;
- изменить интервал отправки.

Интервал подписки можно задавать от `1` до `120` минут. Если пользователь введёт
меньше `1`, бот поставит `1`. Если больше `120`, бот поставит `120`. Если ввести
не число, используется значение по умолчанию `10`.

Фоновая задача проверяет подписки примерно раз в минуту и отправляет посты, если
для подписки наступило время следующей отправки.

### История поиска

Бот сохраняет историю поисковых запросов пользователя.

Историю можно открыть:

- кнопкой `История` в главном меню;
- командой `/history`.

Из истории можно быстро повторить старый запрос, нажав на соответствующую кнопку.

### Избранное

Любой найденный пост можно сохранить в избранное кнопкой `В избранное`.

Избранное можно открыть:

- кнопкой `Избранное` в главном меню;
- командой `/favorites`.

В избранном бот показывает:

- ID поста;
- rating;
- score;
- часть тегов;
- кнопку открытия поста на сайте;
- кнопку удаления из избранного.

### Настройки подписи

Через меню `Настройки` можно управлять тем, что бот показывает в подписи под
медиа.

Можно включать и выключать:

- всю подпись целиком;
- поисковый запрос;
- метку автоматической рассылки;
- ID поста;
- score;
- rating;
- tags.

Подпись ограничивается по длине, чтобы не превышать лимиты Telegram.

### Автодополнение тегов

Команда `/tags <запрос>` ищет подсказки тегов через autocomplete API.

Пример:

```text
/tags blon
```

Бот вернёт список подходящих тегов, если API нашёл варианты.

### Поиск по ID

Команда `/id <номер>` получает конкретный пост по ID.

Пример:

```text
/id 1234567
```

Если пост найден, бот отправит его как изображение или видео и добавит кнопки
управления.

### Антиспам

В `config.py` есть настройка `SEARCH_COOLDOWN_SECONDS`. Она задаёт минимальную
паузу между поисковыми запросами одного пользователя.

По умолчанию:

```env
SEARCH_COOLDOWN_SECONDS=3
```

Если поставить `0`, ограничение отключится.

### Короткие callback_data

Telegram ограничивает размер `callback_data` у inline-кнопок. Поэтому длинные
поисковые запросы и названия подписок не кладутся в кнопку напрямую.

Бот сохраняет полный текст в памяти процесса, а в `callback_data` кладёт короткий
токен. Такие токены живут ограниченное время и исчезают после перезапуска бота.
Если пользователь нажмёт старую кнопку после перезапуска, бот попросит открыть
раздел заново.

## Команды

### `/start`

Показывает приветствие и главное меню.

### `/search <теги>`

Выполняет поиск по тегам.

Пример:

```text
/search genshin_impact blue_eyes
```

### `/tags <запрос>`

Ищет подсказки тегов.

Пример:

```text
/tags hair
```

### `/id <post_id>`

Открывает конкретный пост по ID.

Пример:

```text
/id 1234567
```

### `/blacklist`

Открывает меню blacklist.

### `/subscriptions`

Открывает меню подписок.

### `/history`

Показывает последние поисковые запросы.

### `/favorites`

Показывает избранные посты.

### `/settings`

Открывает настройки подписи.

## Установка

Требуется Python 3.11 или новее.

Установить зависимости:

```powershell
pip install -r requirements.txt
```

Создать `.env` на основе `.env.example`:

```env
BOT_TOKEN=your_telegram_bot_token
API_USER_ID=your_rule34_api_user_id
API_KEY=your_rule34_api_key
SEARCH_COOLDOWN_SECONDS=3
```

Запустить бота:

```powershell
python bot.py
```

Также можно запускать через `rule34.bat`.

## Переменные окружения

### `BOT_TOKEN`

Токен Telegram-бота.

### `API_USER_ID`

User ID для Rule34 API.

### `API_KEY`

API key для Rule34 API.

### `SEARCH_COOLDOWN_SECONDS`

Пауза между поисками одного пользователя в секундах. По умолчанию `3`.

## База данных

Бот использует SQLite-файл `bot_data.db`.

Основные таблицы:

- `users` - последний запрос пользователя;
- `blacklist` - пользовательские blacklist-теги;
- `subscriptions` - подписки и интервалы;
- `user_settings` - настройки подписи;
- `search_history` - история запросов;
- `favorites` - избранные посты;
- `sent_posts` - ID уже отправленных постов;
- `schema_migrations` - заготовка под миграции схемы.

## Тесты

Запуск тестов:

```powershell
python -B -m unittest discover -s tests
```

## Важные примечания

- Бот требует заполненные `BOT_TOKEN`, `API_USER_ID` и `API_KEY`.
- Если обязательных переменных нет, бот пишет ошибку в лог и не запускается.
- Старые inline-кнопки могут перестать работать после перезапуска процесса.
- Уже отправленные пользователю посты больше не должны повторяться при следующих поисках.
