# Night Fun Drug Bot TG

![Visitor Count](https://vercel.app)

Telegram-бот для поиска и просмотра медиа через Rule34 API. Бот поддерживает поиск по тегам, легкий глобальный рандом, blacklist, историю, избранное, галереи, подписки, временную паузу подписок и устойчивую отправку медиа с fallback-ссылками.

Бот рассчитан на совершеннолетних пользователей.

## Возможности

- Поиск постов по тегам через `/search <tags>` или обычное текстовое сообщение.
- Кнопка `Еще` для продолжения текущего поиска без повторов.
- Легкая команда `/random` и кнопка рандома: случайная картинка без поисковых тегов, только с учетом blacklist.
- Исключение уже отправленных пользователю `post_id` через таблицу `sent_posts`.
- Пользовательский blacklist тегов.
- История последних поисков.
- Избранное как меню:
  - `Галерея`
  - `Список`
  - `Найти по тегу`
- Кнопка `Все теги` под постом, которую можно включать и выключать в настройках описания.
- Подписки на поисковые запросы с настраиваемым интервалом.
- Временная пауза всех подписок с автоматическим продолжением после паузы.
- Кнопка быстрого возобновления подписок.
- Новые подписки, добавленные во время паузы, тоже ждут окончания паузы.
- Главное меню показывает, сколько осталось до окончания паузы подписок.
- Локальный SQLite-кэш постов, подписок и медиа-ссылок.
- Atomic claim для фоновой обработки подписок, чтобы не отправлять дубли.
- WAL, `busy_timeout` и `foreign_keys` для SQLite.
- Отправка медиа с fallback-цепочкой `file_url -> sample_url -> preview_url`.
- Параллельная обработка апдейтов и фоновые задачи для долгих поисков, чтобы callback-кнопки не залипали.
- Отдельные файлы логов для `INFO` и `WARNING+`.
- Тесты для API-пагинации, БД, подписок, избранного, Markdown escaping, отправки медиа и callback-flow.

## Настройка

Создайте `.env` рядом с `bot.py` по примеру `.env.example`:

```env
BOT_TOKEN=your_telegram_bot_token
API_USER_ID=your_rule34_api_user_id
API_KEY=your_rule34_api_key
SEARCH_COOLDOWN_SECONDS=3
DB_PATH=bot_data.db
```

Обязательные переменные:

- `BOT_TOKEN` - токен Telegram-бота.
- `API_USER_ID` - Rule34 API user id.
- `API_KEY` - Rule34 API key.

Опциональные переменные:

- `SEARCH_COOLDOWN_SECONDS` - задержка между пользовательскими поисками.
- `DB_PATH` - путь к SQLite-базе.

## Команды

- `/start` - главное меню.
- `/search <tags>` - поиск по тегам.
- `/random` - случайная картинка без тегов, с учетом blacklist.
- `/blacklist` - управление blacklist.
- `/subscriptions` - управление подписками.
- `/history` - история поисков.
- `/favorites` - меню избранного.
- `/settings` - настройки описания и подписок.
- `/tags <query>` - автодополнение тегов.
- `/id <post_id>` - открыть пост по ID.

## Рандом

Обычный поиск по тегам использует интерактивную выдачу и кнопку `Еще`.

Глобальный рандом работает отдельно и легче:

- `limit = 50`
- случайный `pid` от `0` до `200`
- максимум `3` попытки
- timeout `15` секунд на запрос
- blacklist применяется
- уже отправленные пользователю посты исключаются через `sent_posts`

Это снижает вероятность таймаутов по сравнению с тяжелым сканированием больших страниц.

## Избранное

`/favorites` открывает меню с количеством сохраненных постов:

- `Галерея` - основной способ просмотра: один пост на экран, навигация назад/вперед.
- `Список` - постраничная навигация по сохраненным постам.
- `Найти по тегу` - поиск внутри избранного по одному тегу.

Галерея и список используют все сохраненные посты пользователя. Избранное внутри подписок тоже не ограничивается 50 постами.

## Подписки

Подписка хранит поисковый запрос и интервал проверки. Фоновая задача периодически выбирает due-подписки, атомарно забирает их через `processing_token`, отправляет новый пост и освобождает claim.

Для подписок используется локальный пул постов:

1. Бот запрашивает до 1000 постов из Rule34 API.
2. Новые посты добавляются в `subscription_cache`, старые не удаляются.
3. Перед отправкой бот исключает уже отправленные пользователю `sent_posts`.
4. Если API временно не отвечает, бот может использовать старый кэш.

Подписки можно временно остановить в настройках. Пауза хранится на уровне пользователя, поэтому новые подписки, добавленные во время паузы, тоже начнут работать только после ее окончания.

## SQLite

База создается автоматически при старте. По умолчанию используется `bot_data.db`.

Основные таблицы:

- `users`
- `blacklist`
- `subscriptions`
- `search_history`
- `favorites`
- `sent_posts`
- `subscription_posts`
- `subscription_cache`
- `post_cache`
- `user_settings`

Локальные файлы базы и WAL/SHM не должны попадать в Git.

## Логи

Логи пишутся в `logs/`:

- `logs/info.log` - только `INFO`.
- `logs/warnings.log` - `WARNING`, `ERROR`, `CRITICAL`.

Telegram bot token маскируется formatter-ом логов.
