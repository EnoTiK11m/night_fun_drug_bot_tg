# Night Fun Drug Bot TG

[![CI](https://github.com/EnoTiK11m/night_fun_drug_bot_tg/actions/workflows/ci.yml/badge.svg)](https://github.com/EnoTiK11m/night_fun_drug_bot_tg/actions/workflows/ci.yml)
[![Python 3.11](https://img.shields.io/badge/Python-3.11%2B-3776AB?logo=python&logoColor=white)](https://www.python.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

Telegram-бот для поиска, просмотра и организации медиа из [Rule34](https://rule34.xxx/) по тегам. Поддерживает одиночную выдачу, альбомы, избранное, коллекции, blacklist и автоматические подписки.

> [!WARNING]
> Проект предназначен только для совершеннолетних пользователей (18+). Контент поступает из стороннего сервиса и не хранится в репозитории. Владелец запущенного экземпляра самостоятельно отвечает за ограничение доступа и соблюдение применимых правил.

## Возможности

### Поиск и галереи

- поиск по тегам, ID поста и случайная выдача;
- галереи до 10 элементов с переходом между страницами;
- сортировка по новизне, популярности или случайным образом;
- фильтры по рейтингу, типу медиа, ориентации и минимальному разрешению;
- выбор качества: `auto`, `preview`, `sample` или `original`;
- исключение уже просмотренных постов из повторной выдачи;
- автодополнение тегов;
- конструктор запросов с обязательными и исключаемыми тегами;
- сохранённые поисковые пресеты с собственными фильтрами;
- кнопка похожей подборки и быстрые действия с тегами;
- корректная сборка Telegram-альбома при наличии GIF: используется статичное preview, а несовместимое медиа пропускается.

### Персонализация

- постоянная компактная клавиатура с основными действиями;
- настройка состава подписей к изображениям;
- управление максимальным размером загружаемого файла;
- спойлеры для всех медиа или только постов с rating `explicit`;
- история запросов и личная статистика;
- объяснение, какой blacklist-тег заблокировал пост.

### Избранное и коллекции

- добавление постов в избранное;
- просмотр избранного списком или галереей;
- пользовательские коллекции;
- заметки к сохранённым постам;
- отдельная очередь «Посмотреть позже» с автоматическим сроком хранения;
- поиск по тегам и тексту заметок;
- экспорт избранного или отдельной коллекции в ZIP.

### Blacklist

- постоянные и временные исключения;
- добавление и удаление нескольких тегов одной командой;
- готовые пресеты;
- импорт и экспорт списка;
- поиск похожих тегов через автодополнение.

### Подписки и надёжность

- периодическая доставка новых постов по сохранённым запросам;
- отдельные фильтры rating, типа, ориентации, разрешения, качества и blacklist для каждой подписки;
- накопительный дайджест: от пяти постов либо не реже одного раза в шесть часов;
- настраиваемый интервал и временная пауза подписок;
- кэширование результатов и защита от повторной отправки;
- backoff при отсутствии новых постов;
- очередь неудачных доставок и ручной повтор администратором;
- обработка Telegram rate limits и резервные media URL;
- локальное скачивание и загрузка файла, если Telegram не может получить его по URL;
- проверка MIME, сигнатуры и размера изображения, защита от опасных адресов и перенаправлений.

### Рекомендации и массовые действия

- рекомендации на основе частых тегов в избранном без передачи профиля внешнему AI;
- исключение нежелательных тегов из будущих рекомендаций;
- сохранение всей поисковой галереи в избранное или новую коллекцию;
- создание пресета или подписки прямо из результатов галереи;
- просмотр занимаемых пользовательских записей и очистка старой истории.

## Быстрый запуск

### 1. Получите исходный код

```bash
git clone https://github.com/EnoTiK11m/night_fun_drug_bot_tg.git
cd night_fun_drug_bot_tg
```

### 2. Создайте виртуальное окружение

Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Linux и macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### 3. Настройте окружение

Скопируйте пример конфигурации:

```powershell
Copy-Item .env.example .env
```

Для Linux и macOS:

```bash
cp .env.example .env
```

Заполните как минимум три обязательные переменные:

```env
BOT_TOKEN=your_telegram_bot_token
API_USER_ID=your_rule34_api_user_id
API_KEY=your_rule34_api_key
```

Токен Telegram создаётся через [@BotFather](https://t.me/BotFather). Данные Rule34 API доступны в настройках аккаунта сервиса.

### 4. Запустите бота

```bash
python bot.py
```

SQLite-база и необходимые таблицы создаются автоматически при первом запуске.

## Конфигурация

Все параметры читаются из переменных окружения или файла `.env`.

| Переменная | Обязательная | Значение по умолчанию | Назначение |
| --- | :---: | --- | --- |
| `BOT_TOKEN` | да | — | Токен Telegram-бота |
| `API_USER_ID` | да | — | ID пользователя Rule34 API |
| `API_KEY` | да | — | Ключ Rule34 API |
| `SEARCH_COOLDOWN_SECONDS` | нет | `3` | Пауза между пользовательскими поисками |
| `SUBSCRIPTION_CHECK_INTERVAL_SECONDS` | нет | `120` | Частота проверки подписок; минимум 30 секунд |
| `SUBSCRIPTION_MAX_POSTS_PER_USER_PASS` | нет | `45` | Максимум обрабатываемых подписок пользователя за проход, от 1 до 45 |
| `DB_PATH` | нет | `bot_data.db` | Путь к SQLite-базе |
| `ADMIN_USER_IDS` | нет | пусто | Telegram user ID администраторов через запятую |
| `ALLOWED_USER_IDS` | нет | пусто | Разрешённые пользователи личных чатов через запятую |
| `ALLOWED_CHAT_IDS` | нет | пусто | Разрешённые Telegram chat ID через запятую |
| `ALLOW_GROUP_CHATS` | нет | `false` | Разрешить работу в группах |

Если `ALLOWED_USER_IDS` не задан, бот доступен всем пользователям в личных чатах. Группы запрещены по умолчанию. Пользователи из `ADMIN_USER_IDS` всегда получают доступ.

Не добавляйте `.env`, базы, логи и резервные копии в Git. Эти пути уже исключены через [`.gitignore`](.gitignore).

## Запуск в Docker

Создайте `.env`, затем выполните:

```bash
docker compose up -d --build
docker compose logs -f bot
```

Compose хранит данные вне контейнера:

- `./data` — SQLite-база;
- `./logs` — журналы приложения.

Остановка:

```bash
docker compose down
```

Не запускайте несколько экземпляров приложения с одной SQLite-базой.

## Команды бота

### Пользовательские команды

| Команда | Назначение |
| --- | --- |
| `/start` | Установить постоянную клавиатуру и открыть стартовый экран |
| `/search <tags>` | Найти один пост по тегам |
| `/random` | Получить случайный пост |
| `/gallery <tags>` | Получить подборку до 10 элементов; `random` — случайная подборка |
| `/id <post_id>` | Найти пост по ID |
| `/tags <query>` | Найти подходящие названия тегов |
| `/blacklist` | Открыть меню blacklist |
| `/blacklist add <tags>` | Добавить один или несколько тегов |
| `/blacklist remove <tags>` | Удалить один или несколько тегов |
| `/whyblocked <post_id или tags>` | Показать совпадения с blacklist |
| `/favorites` | Открыть избранное |
| `/collections` | Управлять коллекциями избранного |
| `/presets` | Управлять поисковыми пресетами |
| `/recommendations` | Получить рекомендации по избранному |
| `/later` | Открыть очередь «Посмотреть позже» |
| `/storage` | Показать использование пользовательского хранилища |
| `/history` | Показать историю поиска |
| `/stats` | Показать личную статистику |
| `/subscriptions` | Управлять автоматическими подписками |
| `/settings` | Настроить подписи, галереи и качество медиа |

После первого `/start` основные действия доступны на постоянной клавиатуре без повторного открытия стартового меню.

### Команды администратора

| Команда | Назначение |
| --- | --- |
| `/health` | Проверить Rule34 API, SQLite, фоновые задачи и свободное место |
| `/adminstats` | Показать runtime-метрики и статистику базы |
| `/retry_failed` | Повторить до 20 неудачных доставок подписок |
| `/restart` | Завершить процесс с кодом перезапуска для внешнего launcher |

Административные команды доступны только ID из `ADMIN_USER_IDS`. Для автоматического возобновления после `/restart` необходим `rule34.bat`, Docker с restart policy или другой process manager.

## Данные, логи и резервное копирование

По умолчанию база хранится в `bot_data.db`. Приложение включает WAL, foreign keys и busy timeout для безопасной конкурентной работы в рамках одного процесса.

Создание согласованной резервной копии локальной базы:

```bash
python scripts/backup_sqlite.py --db bot_data.db --output-dir backups
```

Для Docker:

```bash
python scripts/backup_sqlite.py --db data/bot_data.db --output-dir backups
```

Основные логи находятся в `logs/`:

- `info.log` — рабочие события и heartbeat;
- `warnings.log` — предупреждения;
- `errors.log` — ошибки и traceback;
- `startup_output.log` и `startup_errors.log` — вывод Windows launcher до настройки logging.

Приложение ротирует рабочие журналы. Практические рекомендации по эксплуатации находятся в [`docs/PRODUCTION.md`](docs/PRODUCTION.md).

## Ограничения

- Бот использует long polling и рассчитан на один активный процесс с одной SQLite-базой.
- Доступность поиска и файлов зависит от Rule34 API и его CDN.
- Telegram не поддерживает GIF как animation внутри media group. В обычной галерее бот использует статичное preview; в режиме анимаций GIF отправляются отдельно.
- Дайджест отправляется после накопления пяти постов или через шесть часов; его можно запросить вручную в меню подписок.
- ZIP-экспорт включает только статичные изображения поддерживаемых форматов и ограничен размером Telegram.
- Проект не предоставляет встроенную проверку возраста — доступ должен контролировать оператор экземпляра.
